Démonstration SAE23 – Application Vendée Globe

Cette page montre le principe de mon projet SAE23 : une application qui récupère des données depuis une base, les expose avec une API REST en JSON, puis les affiche dans une interface web.

Objectif du projet

L’objectif de la SAE23 était de créer une application complète autour des données du Vendée Globe. Le projet reliait une base de données, un serveur Java, une API REST et une interface web.

Sur GitHub Pages, je ne peux pas exécuter PostgreSQL, Java ou Tomcat. Cette page est donc une démonstration statique qui reproduit le principe visuel et logique du projet.

PostgreSQL SQL Java Tomcat API REST JSON HTML CSS JavaScript

Architecture de l’application

1. PostgreSQL Base de données avec les tables skipper, bateau et classement.
2. SQL Requêtes pour lire, trier et filtrer les données.
3. Java/Tomcat Serveur qui récupère les données et prépare les réponses.
4. API REST Points d’accès comme /classement, /skippers et /bateaux.
5. Web JavaScript récupère le JSON et remplit les tableaux HTML.
Idée principale : les données ne sont pas écrites à la main dans la page. Elles viennent normalement de la base via une API, puis JavaScript les affiche dynamiquement.

Exemple du fonctionnement JavaScript

Dans le vrai projet, JavaScript utilisait fetch pour appeler l’API REST. Ici, les données JSON sont simulées directement dans la page pour que la démonstration fonctionne sur GitHub Pages.

fetch("/api/classement")
  .then(response => response.json())
  .then(data => afficherTableau(data));

// Résultat attendu : les données JSON sont affichées dans un tableau HTML.

Ce que cette preuve montre

Cette démonstration montre que j’ai compris la logique d’une application web complète : les données sont stockées dans une base, récupérées par un serveur, envoyées au format JSON, puis affichées par JavaScript dans une page web.

Elle illustre le lien avec la compétence Programmer : utiliser des outils informatiques, comprendre une architecture web, manipuler des données et produire une interface exploitable.
