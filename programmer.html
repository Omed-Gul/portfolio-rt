Compétence 3 – Créer des outils et applications informatiques pour les R&T

Niveau 1 : S’intégrer dans un service informatique

Preuve principale : SAE23 – Application Vendée Globe

Dans cette SAE, j’ai travaillé sur une application web complète autour des données du Vendée Globe. Le projet était organisé en plusieurs parties : base de données, requêtes SQL, backend avec API REST, puis frontend web.

Ce projet m’a permis de comprendre l’architecture d’une application complète : une base PostgreSQL stocke les données, un serveur Java/Tomcat fournit des réponses JSON, et une interface web utilise HTML, CSS, JavaScript et Bootstrap pour afficher les résultats.

Éléments travaillés :

PostgreSQL SQL Java Tomcat API REST JSON HTML CSS JavaScript Bootstrap

Preuves complémentaires : SAE13 – Données audio et image

La SAE13 complète cette compétence car elle montre ma capacité à manipuler des données par programme. Dans l’activité audio, j’ai utilisé MATLAB pour lire un fichier audio, tracer le signal, segmenter des phonèmes, reconstruire un mot et exporter un nouveau fichier audio.

Dans l’activité image, j’ai utilisé MATLAB pour lire une image, la convertir en niveaux de gris, extraire ou déplacer des zones, corriger l’image avec des transformations et enregistrer le résultat final.

Éléments travaillés :

MATLAB Audio Image Segmentation Transformation de données

Apprentissages critiques liés

  • AC13.01 : Utiliser un système informatique et ses outils.
  • AC13.02 : Lire, exécuter, corriger et modifier un programme.
  • AC13.03 : Traduire un algorithme dans un langage et un environnement donné.
  • AC13.04 : Connaître l’architecture et les technologies d’un site Web.
  • AC13.05 : Choisir les mécanismes de gestion de données adaptés.
  • AC13.06 : S’intégrer dans un environnement de développement et de travail collaboratif.

Ce que j’ai appris

J’ai compris qu’une application informatique ne se résume pas au code visible dans une page web. Elle repose sur plusieurs couches : la base de données, le backend, l’API, le format JSON et le frontend. J’ai aussi appris à organiser mes fichiers, tester progressivement, corriger les erreurs et relier plusieurs technologies ensemble.
